Converting, Downloading, & Hosting

  • Sometimes you want to use a particular font, but it's not available on Google Fonts, or you can't find a web type version.
  • One possible solution is to use FontSquirrel to convert your fonts and host them yourself.

Currently this element has no special font attached, but let's add one!

  • Find your font on FontSquirrel or wherever it is you find fonts (we will talk about DaFont later..........)
  • Go to the Generator page
  • Run the generator & download the results
  • Copy the fonts into your fonts directory & the CSS into your CSS file.
  • Important! Make sure you're accurately pointing your CSS rule to the correct directory!!!
  • Apply your new font to the desired element!

Using CSS to make type pop!

  • There are a number of things we can do to help out our web typography, without the help of Photoshop or another editor.
  • Detail is key in successful web typography.
  • Font Stacks
  • Built-in Categories
  • Font Size
  • Line Height
  • Letter Spacing
  • Font stacks allow us to set us a variety of fallbacks in case a webfont doesn't load correctly or the user doesn't have the desired font in their database.
  • There a bunch of pre-rolled font stacks available around the web.
  • Or you can roll your own!

Ideally this font should display in Poppins, but I don't mind seeing it in Raleway or Futura. If it has to, it can display in Arial.

  • A font stack is written in order from most preferred to least preferred.
  • There are a couple default fonts that should be at the end of every font stack.
  • There are some built-in defaults that are used to give handy fallbacks if you want a particular look but aren't sure what font your viewer will have.

I'm going to go ahead and quote here from a great piece in Smashing Mag, way back from 2011:

In print, type as small as 8 points is an ideal compromise between readability and cost, because you have to pay for every millimeter you use. On the Web, you pay nothing for using more space — provided that readers find your copy compelling, of course. What you do pay for is readers finding your compelling copy so hard to read that they click away instead of converting.

Font size also isn't just a question of readability -- it can be used to draw the reader's eye to important content, or to organize the rhythm and flow of a design.

Line height is another one of those things that can mean the difference between easily readable and totally illegible. A good rule of thumb is to keep your line-height property between 1.2em and 1.4em.
